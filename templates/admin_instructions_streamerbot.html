StreamerBot Integration

Connect OBS-TV-Animator to StreamerBot to automatically trigger Smart TV animations based on stream events like follows, donations, subscriptions, raids, and chat commands.

Prerequisites: StreamerBot (free), Smart TV already connected. Complete Getting Started first →

StreamerBot Setup

1

Download StreamerBot

If you don't have StreamerBot yet:

  1. Visit: https://streamer.bot/
  2. Download and install the latest version (free)
  3. Connect to your streaming platform (Twitch/YouTube)
  4. Set up basic platform integration
2

Test OBS-TV-Animator Connection

First, verify the connection is working by testing in your browser:

Test URLs (open in browser):
http://localhost:5000/trigger?animation=anim1.html
http://localhost:5000/trigger?animation=anim2.html
http://localhost:5000/trigger?animation=brb.html

If these work in your browser, they'll work in StreamerBot too!

Success: You should see a JSON response like:
{"success": true, "current_animation": "anim1.html", ...}

Creating StreamerBot Actions

StreamerBot uses "Actions" to respond to events. The recommended method uses C# Execute Code for direct WebSocket communication.

New Recommended Method!

The C# WebSocket method provides better performance, reliability, and error handling compared to HTTP requests. Use the Manage Files page to copy ready-to-use C# code for each animation.

1

Create New Action

  1. Open StreamerBot
  2. Go to Actions tab
  3. Right-click in the Actions list
  4. Select "Add Action"
  5. Name it (e.g., "TV Animation - Follow")
  6. Click OK
2

Define a Trigger

  1. In your new action, right-click in the Triggers box
  2. Select Add
  3. Choose the trigger you would like (e.g., Twitch → Raid, Twitch → Follow, etc.)
  4. Configure any trigger-specific variables or settings
  5. Click OK
3

Add Execute C# Code Sub-Action

  1. In your new action, right-click in the Sub-Actions area
  2. Navigate: Add → Core → C# → Execute C# Code
  3. Copy the C# code from the Manage Files page
  4. Paste it into the C# code editor
  5. Click Compile & Save
Example C# Code:
using System;
public class CPHInline
{
    public bool Execute()
    {
        string json = "{\"action\":\"trigger_animation\",\"animation\":\"anim1.html\"}";
        CPH.WebsocketSend(json, 0);
        return true;
    }
}
Pro Tip: Use the "StreamerBot C#" button next to each animation in Manage Files for file-specific code.
Why this works: StreamerBot's Execute C# Code action uses CPH.WebsocketSend() to communicate directly with OBS-TV-Animator via WebSocket. This provides instant triggering, better error handling, and bidirectional communication.

3

Add Fetch URL Sub-Action

  1. In your new action, right-click in the Sub-Actions area
  2. Navigate: Add → Core → Network → Fetch URL
  3. Configure the Fetch URL action:
Fetch URL Configuration:
URL: http://localhost:[PORT]/trigger?animation=anim1.html
Variable Name: triggerResponse (can be any name)
Method: GET (default)

Pro Tip: The variable name doesn't affect functionality, it's just for storing the response.
Why this works: StreamerBot's Fetch URL action makes a simple GET request to your OBS-TV-Animator server. The animation parameter in the URL tells the server which animation to display.

Advanced StreamerBot Features

Random Animation Selection

Create actions that randomly pick from your available animations:

  1. Create a new action: Right-click in Actions → Add → Give it a name like "Random Animation"
  2. Add trigger: Right-click in Triggers → Select your event (Follow, Donation, etc.)
  3. Add sub-action: Right-click Sub-Actions → Core → Execute Code (C#)
  4. Use the C# code from Manage Files, but modify the animation name to use StreamerBot's random selection

Pro Tip: Use StreamerBot's built-in random functions or create multiple actions with different animations and use random sub-action execution.

Conditional Animations Based on Event Data

Scale animation intensity based on donation amounts, subscriber months, etc.:

  1. Create separate actions for different thresholds (e.g., "Small Donation", "Big Donation")
  2. Set up conditional logic using StreamerBot's If/Else sub-actions
  3. Use variables like %donationAmount%, %subMonths%, or %viewerCount%
  4. Route to different animation sub-actions based on the conditions

Example Setup: If donation ≥ $100 → anim4.html, if ≥ $10 → anim2.html, else → anim1.html

Timed Events & Automation

Set up automatic animations using StreamerBot's built-in scheduling:

  • Hourly reminders: Use Timed Actions to trigger follow/subscribe animations
  • Break timers: Create actions triggered by scene changes to auto-show BRB screen
  • Stream milestones: Use viewer count triggers for celebration animations
  • Game changes: React to OBS scene or game detection events

Setup: Go to Actions → Timed Actions to create recurring triggers, or use Events → OBS for scene-based automation.

Advanced Animation Control

Fine-tune your animation behavior with optional parameters:

  • Instant triggering: Add "instant": true to your JSON for immediate response
  • Force refresh: Use "force_refresh": true to ensure TV display updates
  • Source tracking: Add "source": "custom_name" to identify different trigger types

How to use: When copying C# code from Manage Files, you can modify the JSON string to include these parameters for enhanced control.

Quick Reference: All animations include copy buttons for both basic and HTTP integration. Use the StreamerBot C# button for instant, copy-paste setup.

Testing & Troubleshooting

Test Actions

Always test your actions before going live:

  1. Right-click your action in StreamerBot
  2. Select "Test Trigger"
  3. Watch your Smart TV for the animation change
  4. Check StreamerBot logs for any errors

HTTP Request Failures

  • Verify OBS-TV-Animator is running and accessible
  • Check the IP address and port (8080)
  • Ensure JSON format is correct in request body
  • Look at StreamerBot logs for specific error messages

Events Not Triggering

  • Check StreamerBot connection to your streaming platform
  • Verify event triggers are properly configured
  • Test with manual triggers first
  • Check if there are permission or cooldown restrictions

C# Execution Failures

When C# Execute Code sub-actions fail to trigger animations:

  • Check StreamerBot logs: Go to Logs tab in StreamerBot for C# compilation or runtime errors
  • Verify WebSocket connection: Look for connection establishment messages in StreamerBot logs
  • Check Docker/OBS-TV-Animator logs: View container logs to confirm WebSocket handshake completion
  • Test JSON syntax: Ensure the JSON string in your C# code is properly formatted (no missing quotes or commas)
  • Confirm network connectivity: Make sure StreamerBot can reach your Docker container (same network, correct port)
  • Debug step-by-step: Add CPH.LogInfo("Debug message"); to your C# code to track execution flow
Quick Test: Try the basic HTTP method first - if it works but C# doesn't, the issue is likely in your C# code or StreamerBot's WebSocket setup.

Complete C# Code Template

Here's a comprehensive C# template showing all available parameters and structure:

{% raw %}using System;
public class CPHInline
{
    public bool Execute()
    {
        // Animation file to trigger (change this to your desired animation)
        string animationFile = "anim1.html";
        
        // Complete JSON with all available parameters
        string json = $@"{{
            ""action"": ""trigger_animation"",
            ""animation"": ""{animationFile}"",
            ""instant"": true,
            ""force_refresh"": true,
            ""source"": ""streamerbot_websocket""
        }}";
        
        // Send WebSocket message to OBS-TV-Animator
        // Note: Change "0" to match your WebSocket connection index if you have multiple WebSocket servers
        CPH.WebsocketSend(json, 0);
        
        // Optional: Log success message
        CPH.LogInfo($"Triggered animation: {animationFile}");
        
        return true;
    }
}{% endraw %}

Usage: Copy this template into StreamerBot's Execute Code (C#) sub-action

Customize: Change animationFile and source values as needed. Remove parameters you don't need.

WebSocket Index: If you have multiple WebSocket servers in StreamerBot, change the 0 in CPH.WebsocketSend(json, 0) to match your OBS-TV-Animator connection index.

Tip: Use the copy buttons in Manage Files for specific animations instead of manually editing this template.
