Getting Started with OBS-TV-Animator

Welcome! This guide will walk you through the basic setup process to get your Smart TV displaying animations from OBS-TV-Animator.

Smart TV Setup

Congratulations!

If you've made it here, you have successfully setup Docker with OBS-TV-Animator. Make sure that your bind mounts were added correctly in the docker-compose.yml or through Docker GUI/Docker Desktop to ensure that files and settings persist across restarts.

1

Test Connection from Computer

First, test that the server is working by opening this URL in your computer's web browser:

http://[YOUR_IP]:[PORT]

You should see an animation playing. If not, check your Docker setup and firewall settings.

Port: Default is 8080, but check your docker-compose.yml for the actual port mapping

Find your IP: Windows: ipconfig | macOS/Linux: ifconfig

2

Connect Your Smart TV

On your Smart TV:

  1. Open the web browser app (available on most modern Smart TVs)
  2. Navigate to: http://[YOUR_IP]:[PORT]
  3. The current animation should auto-play fullscreen
  4. Changes will appear instantly when triggered from the admin panel
3

Test Animation Switching

From the Admin Dashboard:

  1. Go to the Available Media section
  2. Click "Play Now" on any animation
  3. Watch the TV update in real-time!

User Management

Secure your admin panel by managing user accounts and changing default credentials.

Add or Remove Users

To manage user accounts, visit the Manage Users page where you can:

  • Add new admin users with custom usernames and passwords
  • Remove users who no longer need access
  • View login timestamps for security monitoring

Security Recommendations

Important: If you haven't already done so, we recommend changing the default username and password and removing the old admin account.

  • Create a new admin user with a strong, unique password
  • Log in with the new account to test it works
  • Delete the default admin account for security

Note: You cannot delete an account that's currently logged in. Switch to your new account first, then delete the default credentials.

Understanding Animation Types

OBS-TV-Animator supports both HTML animations and video files, each with their own advantages.

HTML Animations

Interactive web-based animations with CSS and JavaScript:

  • Advantages: Lightweight, scalable, interactive, real-time data integration
  • Best for: Text overlays, countdowns, subscriber alerts, dynamic content
  • File types: .html files with embedded CSS/JS

Video Files

Pre-rendered video content for high-quality animations:

  • Advantages: Consistent playback, complex effects, professional quality
  • Best for: Complex animations, transitions, branded content
  • File types: .mp4, .webm, .mov, .avi

Smart TV Best Practices

For optimal performance and compatibility on Smart TV displays, follow these guidelines:

HTML Animation Tips

  • Use web-safe fonts: Arial, Helvetica, sans-serif for maximum compatibility
  • Avoid complex JavaScript: Keep animations simple for better TV browser performance
  • Test responsiveness: Ensure animations work at various screen resolutions
  • Use high contrast: Bold colors and clear text show better on TV screens
  • Minimize network requests: Embed resources when possible to reduce loading times

OBS-TV-Animator Integration

For full system integration, add these lines to your custom HTML animations:

HTML Head Section 
<!-- OBS-TV-Animator Integration -->
<link rel="stylesheet" href="/static/css/ota-integration.css">
Before Closing Body Tag 
<!-- Socket.IO and OTA Integration -->
<script src="https://cdn.socket.io/4.7.2/socket.io.min.js"></script>
<script src="/static/js/ota-integration.js"></script>
<script>
  document.addEventListener('DOMContentLoaded', () => {
    window.otaIntegration = new OTAIntegration('mainElement');
  });
</script>
  • Connection Status: Adds a status dot in the top-right corner
  • Page Refresh: Enables seamless content switching
  • Flash Effects: Visual feedback when animations change
  • WebSocket Integration: Real-time communication with OTA server
  • Replace 'mainElement': Use the ID of your main animation element for flash effects

Optional: Animations work without integration, but these features enhance the user experience.

Performance Guidelines

  • Keep file sizes reasonable: HTML animations under 1MB, videos under 50MB
  • Optimize images: Use compressed formats (WebP, optimized PNG/JPG)
  • Limit animation complexity: Too many moving elements can cause stuttering
  • Test on actual TV hardware: TV browsers may perform differently than desktop

Backup & Data Persistence

Protect your animations, settings, and user accounts during container updates or migrations.

Important Directories

These directories contain all your persistent data and should be backed up:

  • animations/ - Your HTML animation files
  • videos/ - Your uploaded video files
  • data/ - User accounts, settings, and thumbnails

Migration Process

To migrate or reinstall your container while preserving data:

  1. Copy the animations/, videos/, and data/ directories to a safe location
  2. Install the new container or move to a new system
  3. Restore the directories to the same locations
  4. Update your docker-compose.yml to map these directories correctly
  5. Start the container - all settings, accounts, and files will be preserved

Tip: Regular backups of these directories ensure you never lose your custom content or configuration.

Network Requirements

Same Network Required

Your computer running Docker and your Smart TV must be on the same local network.

Common network issues:

  • TV connected to guest WiFi network
  • Computer on wired connection, TV on different WiFi
  • VPN blocking local network access

Firewall Configuration

Ensure your chosen port (default 8080) is allowed through your firewall:

  • Windows: Windows Defender Firewall may block the port
  • macOS: Check System Preferences → Security & Privacy → Firewall
  • Router: Most home routers allow local network traffic by default
